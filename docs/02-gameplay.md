# 02 — Gameplay

## Core Gameplay Loop

1. Arrive at the office for the night shift.
2. Receive a batch of undeliverable mail.
3. Inspect each letter.
4. Read the contents.
5. Check envelope details:
   - Sender
   - Recipient
   - Address
   - Date
   - Stamp
   - Postmark
   - Condition
   - Handwriting
   - Special markings
6. Cross-reference with:
   - Town map
   - Address registry
   - Death records
   - Missing persons file
   - Postal policy manual
   - Previous letters
   - Newspaper archive
7. Decide what to do:
   - Deliver
   - Archive
   - Destroy
   - Return to sender
   - Flag for investigation
   - Hide from the office
8. Stamp and sort the letter.
9. Receive consequences:
   - New letters unlock
   - Office changes
   - Character files update
   - Map changes
   - Hidden lore appears
   - Strange events happen
10. End the shift.
11. Return the next night with new rules, new letters, and deeper contradictions.

The loop should be simple enough to understand immediately, but deep enough that players keep thinking:

> "Wait, this letter changes what I thought the last letter meant."

## Letter Inspection

Each letter has inspectable components.

Envelope details:

- Recipient name
- Sender name
- Address
- Postmark
- Date
- Stamp type
- Ink color
- Seal condition
- Damage
- Handwriting
- Hidden markings
- Weight
- Smell or stain description

Letter content:

- Body text
- Crossed-out words
- Postscript
- Codes
- Unusual phrasing
- Repeated symbols
- Contradictions

Tools:

- Letter opener
- Magnifying glass
- UV lamp
- Heat lamp
- Audio scanner
- Postal code book
- Date stamp verifier
- Red string board (optional)
- Case files

## Sorting Decisions

Each letter can be sorted into different outcomes. Possible bins:

- **Deliver**
- **Archive**
- **Return to Sender**
- **Destroy**
- **Flag for Review**
- **Hold**
- **Personal Drawer**

Each decision affects the game. Examples:

- Delivering a letter may change a character's fate.
- Destroying a letter may erase evidence.
- Archiving a letter may preserve memory but anger the office.
- Returning a letter may cause it to come back altered.
- Hiding a letter may unlock secret storylines.

## Rule Changes

Each shift introduces or changes official rules. Example rules:

- "All letters addressed to deceased recipients must be destroyed."
- "Letters dated after the current year must be archived without reading."
- "Letters containing the phrase 'I remember' must be flagged."
- "Mail from Briar Lane must not be delivered."
- "Personal references to the clerk are to be ignored."
- "Do not process mail without a valid postmark."
- "Do not open black-sealed envelopes."
- "Do not read aloud."

Rules should sometimes conflict with morality or common sense. The player can obey or disobey.

## Cross-Referencing

The player must compare letters with records. Reference systems:

- Address registry
- Death records
- Newspaper archive
- Town map
- Missing persons board
- Employee handbook
- Postal law binder
- Previous letters
- File cabinets

Example puzzle:

A letter is dated 1964 from Jonah Bell to Ruth Bell. The death records say Jonah died in 1958. The address registry says Ruth Bell's house was demolished in 1960. The map shows the house still exists — if the player archived Ruth's previous letter.

The player must decide whether to deliver the letter, archive it, or destroy it.

## Office Compliance System (Pressure Without a Sanity Meter)

Avoid a generic "sanity meter." Instead, use an **Office Compliance System**: the office tracks whether the player follows policy.

Metrics:

- Compliance
- Memory contamination
- Unauthorized deliveries
- Destroyed evidence
- Hidden letters
- Contradictions reported
- Personal exposure

The office responds with:

- Warning memos
- Locked drawers
- Changed rules
- Flickering lights
- Reordered rooms
- Altered letters
- Phone calls from no one
- Performance reviews

The system should feel bureaucratic, not gamey.

## Hidden Memory System

Certain letters restore pieces of erased memory. The player can discover:

- Names
- Places
- Events
- Childhood objects
- Sounds
- Drawings
- Photos
- Phrases

As memories return, the office becomes less stable. This can affect:

- Available choices
- Ending path
- Secret rooms
- Letter contents
- The town map
- Player identity reveals

## Interaction Design

The player should interact with letters physically. Interaction flow:

1. Pick up envelope.
2. Rotate envelope.
3. Inspect front and back.
4. Use letter opener.
5. Pull out letter.
6. Read letter.
7. Compare with records.
8. Choose stamp.
9. Stamp envelope.
10. Place in bin.

Important:

- Stamping should feel satisfying.
- Paper should sound good.
- Opening letters should feel tactile.
- Sorting should be fast after the player understands it.
- UI should never become annoying.

Possible stamps:

- DELIVER
- ARCHIVE
- RETURN
- DESTROY
- REVIEW
- UNDELIVERABLE
- DECEASED
- INVALID ADDRESS
- MEMORY CONTAMINATION (late-game)
- CONFESS (secret)

## Puzzle Design

Puzzles should be based on observation and contradiction.

| Puzzle type | Description |
| --- | --- |
| Address puzzle | Letter address does not exist until another letter is archived. |
| Date puzzle | Letter was sent after recipient died. |
| Name puzzle | Same handwriting appears under different names. |
| Code puzzle | First letters of each line form a hidden phrase. |
| Map puzzle | Street names correspond to file cabinet codes. |
| Stamp puzzle | Wrong stamp creates an unexpected future letter. |
| Rule conflict puzzle | Office rule says destroy, but story evidence says preserve. |
| Hidden ink puzzle | UV light reveals a message. |
| Audio puzzle | Phone call repeats a knocking pattern from a letter. |
| Archive puzzle | Re-reading old letters after office changes reveals altered text. |

Puzzle design rule:

> The player should feel clever, not tricked.
